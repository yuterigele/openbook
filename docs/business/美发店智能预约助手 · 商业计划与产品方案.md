> **日期**：2026-06-21 | **版本**：v1.0

## 0. 一句话定位

**每天不到两块钱，雇一个 AI 预约助理。**

基于企业微信 + AI 大模型，为中小美发店提供对话式智能预约。顾客在微信里跟 Agent 私聊就能自助预约，理发师彻底摆脱“人工排班客服”的角色，专注手艺。

---

## 1. 项目概述

### 1.1 目标客户

- 中小美发店，1–5 名理发师，夫妻店优先
- 日均预约 5–30 单
- **核心痛点**：老客都有理发师微信，预约全靠手工沟通——打字确认时间、翻排班本、理发中途接电话。理发师既要服务又要当客服，两头耽误

### 1.2 核心价值主张

| 价值 | 说明 |
|------|------|
| **降本** | 替代一名兼职前台，单店年节省约 4 万元（按 3500 元/月计） |
| **增收** | 减少爽约约 30%，档期不冲突提升翻台率约 10% |
| **省心** | 7×24 自动应答，理发师专注手艺，再也不用边剪头边回微信 |

### 1.3 三方角色

| 角色 | 谁 | 主要动作 |
|------|----|---------|
| C 端顾客 | 加了 Agent 企业微信的顾客 | 私聊发起/改约/取消预约，自然语言交互 |
| B 端商户 | 美发店主/店长 | 看排班、收经营数据、管理理发师请假 |
| 平台 | 我们 | 运行 Agent 服务、维护订阅与对账 |

## 2. 痛点与解决方案

| 痛点 | 现状 | 解决方案 |
|------|------|---------|
| 反复确认时间 | 顾客微信问，理发师切屏翻排班本 | Agent 实时查档期，对话中直接确认 |
| 手工排班冲突 | 两人同时段被约 | Agent 实时冲突检测，自动推荐替代方案 |
| 不清楚能做什么服务 | 顾客逐个问，理发师打字解释 | Agent 内置服务菜单，一问便知 |
| 爽约/忘约 | 全凭顾客自觉 | 预约前 2 小时自动微信提醒 |
| 服务被打断 | 理发中途接电话回微信 | 顾客全程跟 Agent 对话，师傅不被打扰 |
| 经营凭感觉 | 没有数据积累 | 经营看板：人次、热门服务、爽约率一目了然 |

## 3. 产品方案

### 3.1 顾客端：微信私聊预约

顾客在微信里直接给 Agent 发消息，自然语言完成全流程。

**典型对话示例：**

```
顾客：你好，我想约明天下午染发

Agent：好的～明天下午 Tony 老师 14:00 可约，Kevin 老师 14:30 可约。
       染发预计 90 分钟，您想约哪位老师？

顾客：Tony 吧，2 点

Agent：已为您预约：
       📅 明天（6月22日）14:00–15:30
       ✂️ 染发
       💇 Tony 老师
       确认无误请回复“确认”

顾客：确认

Agent：✅ 预约成功！明天下午 2 点 Tony 老师等您～
       提前 2 小时我会提醒您的 😊
```

**冲突时自动推荐替代方案：**

```
顾客：约 Tony 明天下午 2 点烫发

Agent：不好意思，明天下午 Tony 老师 14:00–16:00 已有预约。
       您可以考虑：
       · 换 Kevin 老师同时段
       · 或改约 16:30
```

### 3.2 服务项目

| 服务 | 预估时长 | 说明 |
|------|:----:|------|
| 剪发 | 30 分钟 | 基础剪发 |
| 烫发 | 120 分钟 | 冷烫/热烫 |
| 染发 | 90 分钟 | 单色染 |
| 洗吹 | 20 分钟 | 洗发+吹干造型 |
| 护理 | 45 分钟 | 深层护理/头皮护理 |
| 造型 | 30 分钟 | 吹风造型/一次性造型 |
| 其他 | 30 分钟 | 兜底，具体到店确认 |

Agent 根据服务项目自动匹配所需时长，从 30 分钟档期池中智能分配。

### 3.3 商户端：经营看板

商户通过企业微信接收日报数据，也可登录极简 Web 后台查看：

- **今日概览**：预约总数、已完成、爽约数
- **热门服务排行**：哪个服务最走量
- **热门时段分布**：哪个时间段最忙
- **爽约率趋势**：本周/本月对比
- **理发师业绩排行**：谁的单量最高

### 3.4 关键产品规则

**取消策略：**

| 取消时机 | 是否计罚 | 说明 |
|---------|:----:|------|
| 距预约时间 ≥ 2 小时 | 不计 | 自由取消 |
| 距预约时间 < 2 小时 | 计 1 次“晚退订” | 累计 3 次自动加入黑名单 |
| 未取消也未到店（爽约） | 计 1 次“爽约” | 累计 2 次自动加入黑名单 |
| 商户主动取消 / 系统取消 | 不计 | 不影响顾客信用 |

**理发师请假：**

商户可在后台一键设置理发师请假，系统自动处理该时段内的未来预约：
- **改派**：自动匹配同时间段有空且技能匹配的其他理发师
- **取消并通知**：无法改派时取消预约，自动微信通知顾客

**转人工兜底：**

当顾客需求超出 Agent 能力范围（如投诉、退款、复杂议价），Agent 主动标记转人工，商户在后台可见待处理列表，主动联系顾客。

## 4. 企业微信接入方案

### 4.1 接入架构

企业微信自建应用本身仅限企业内部成员使用，外部微信用户无法直接添加。因此采用双层通路：

**当前阶段（开发期，认证前）：微信客服**

- 顾客扫码或点链接直接进入对话，无需加好友
- 适合印在立牌、海报上，作为新客入口

**目标阶段（认证通过后）：客户联系**

- 店员在企业微信上导入老客手机号，一键添加好友
- 老客直接私聊 Agent，体验最自然
- 店员推广话术：“姐，以后约时间直接找这个助手就行，它会自动帮我排好，我剪头的时候不用接电话了～”

两个通路并行运行，消息通过统一回调接口进入 Agent 核心服务。

### 4.2 顾客添加路径

| 路径 | 操作方式 | 适用场景 |
|------|---------|---------|
| 店员主动添加 | 店员导入老客手机号，一键添加并发送欢迎语 | 存量老客，冷启动核心动作 |
| 扫码立牌 | 店内收银台/镜台放置“扫码预约”台卡 | 到店新客、路过散客 |
| 名片推荐 | 店员在个人微信上把 Agent 名片推给顾客 | 已有店员微信但未加 Agent 的顾客 |

## 11. 总结

### 核心优势

1. **成本极轻**：年固定成本仅约 2,350 元，边际成本趋近于零
2. **毛利极高**：保守场景净利润率 93%，乐观场景 99%
3. **定价有竞争力**：19.9 元/月体验价仅为同类产品的 1/12，夫妻店毫无决策压力
4. **场景极度贴合**：直接接管理发师最痛苦的“微信人工排班”，切中刚需
5. **合规路径清晰**：企业微信官方生态，双层通路有冗余保障
6. **可复制性强**：验证单城模型后，可快速复制到全国美发店

> **一句话总结：前期投入不到 3,000 块，跑通后年利润可达数万至数十万。聚焦美发店，用 AI 私聊 Agent 接管理发师最累的微信沟通环节——这是一个极低成本、高毛利、可复制的垂直 SaaS 项目。**
